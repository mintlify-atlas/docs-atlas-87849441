## Introduction

The OrionRides API is a RESTful API built with Express.js that provides programmatic access to the car rental platform. The API allows you to manage users, vehicles, and rental bookings.

## Base URL

All API endpoints are prefixed with `/api`:

```
http://localhost:3000/api
```

In production, replace `localhost:3000` with your deployed server URL.

## API Structure

The API is organized into three main route groups as defined in `app.js:22-24`:

- **Index Routes** (`/api`) - Booking and rental management
- **User Routes** (`/api/user`) - User authentication and profile
- **Vehicle Routes** (`/api/vehicles`) - Vehicle browsing and details

## Response Format

All API responses are returned in JSON format. The API uses standard HTTP response codes:

### Success Responses

```json
{
  "message": "Success message",
  "data": {}
}
```

### Error Responses

Error responses include a descriptive message and optional error details:

```json
{
  "message": "Error description",
  "error": "Detailed error message"
}
```

## HTTP Status Codes

The API uses conventional HTTP status codes:

| Status Code | Description |
|-------------|-------------|
| `200` | Success - Request completed successfully |
| `201` | Created - Resource created successfully |
| `400` | Bad Request - Invalid request parameters |
| `401` | Unauthorized - Authentication required or failed |
| `404` | Not Found - Resource not found |
| `500` | Internal Server Error - Server error occurred |

## Error Handling

All endpoints implement comprehensive error handling. Errors are caught and returned with appropriate status codes and messages. For example, from `index.route.js:81-87`:

```javascript
try {
  // API logic
} catch (error) {
  console.error("Error:", error);
  res.status(500).json({
    message: "Error renting car!",
    error: error.message,
  });
}
```

## CORS Configuration

The API supports Cross-Origin Resource Sharing (CORS) as configured in `app.js:10-17`. Allowed methods include:

- GET
- POST
- PUT
- DELETE
- PATCH
- HEAD

## Content Type

The API accepts and returns JSON data. All requests should include:

```
Content-Type: application/json
```

## Rate Limiting

Currently, the API does not implement rate limiting. This may be added in future versions.

## API Versioning

The current API version is v1. All endpoints use the `/api` prefix without explicit versioning.
