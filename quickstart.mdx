## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the OrionRides repository to your local machine:

    ```bash
    git clone https://github.com/yourusername/orionrides.git
    cd orionrides
    ```

    The project contains two main directories:
    - `client/` - React frontend application
    - `server/` - Express backend API
  </Step>

  <Step title="Set up the server">
    Navigate to the server directory and install dependencies:

    <CodeGroup>
    ```bash npm
    cd server
    npm install
    ```

    ```bash yarn
    cd server
    yarn install
    ```
    </CodeGroup>

    Create a `.env` file in the `server/` directory with the following variables:

    ```bash .env
    # Server Configuration
    PORT=3000

    # Database
    DB_URL=mongodb://localhost:27017/orionrides
    # Or use MongoDB Atlas:
    # DB_URL=mongodb+srv://<username>:<password>@cluster.mongodb.net/orionrides

    # CORS
    CORS_ORIGIN=http://localhost:5173

    # JWT Secret
    JWT_SECRET=your_jwt_secret_key_here
    ```

    <Note>
      Replace `your_jwt_secret_key_here` with a secure random string for production use.
    </Note>
  </Step>

  <Step title="Set up the client">
    In a new terminal, navigate to the client directory and install dependencies:

    <CodeGroup>
    ```bash npm
    cd client
    npm install
    ```

    ```bash yarn
    cd client
    yarn install
    ```
    </CodeGroup>

    Create a `.env` file in the `client/` directory:

    ```bash .env
    # Backend API URL
    VITE_BACKEND_URL=http://localhost:3000
    ```

    <Info>
      The Vite dev server uses this URL for API proxy configuration in `vite.config.js`.
    </Info>
  </Step>

  <Step title="Start the development servers">
    You'll need two terminal windows - one for the server and one for the client.

    **Terminal 1 - Start the server:**

    <CodeGroup>
    ```bash npm
    cd server
    npm start
    ```

    ```bash yarn
    cd server
    yarn start
    ```
    </CodeGroup>

    You should see:
    ```
    MongoDB Connected Successfully
    The app is listening on port 3000
    ```

    **Terminal 2 - Start the client:**

    <CodeGroup>
    ```bash npm
    cd client
    npm run dev
    ```

    ```bash yarn
    cd client
    yarn dev
    ```
    </CodeGroup>

    The Vite dev server will start at `http://localhost:5173`
  </Step>

  <Step title="Verify the installation">
    Open your browser and navigate to:

    ```
    http://localhost:5173
    ```

    You should see the OrionRides homepage. Try the following:

    <Tabs>
      <Tab title="Create an account">
        1. Click on the login/signup button
        2. Register with your email and password
        3. Verify you're redirected to the home page after signup
      </Tab>
      
      <Tab title="Browse vehicles">
        1. Navigate to the vehicles page
        2. Browse available cars
        3. Click on a vehicle to view details
      </Tab>
      
      <Tab title="Test the API">
        Use curl or Postman to test the API endpoints:

        ```bash
        # Get all vehicles
        curl http://localhost:3000/api/vehicles

        # Health check
        curl http://localhost:3000/api
        ```
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="MongoDB connection error">
    **Error:** `MongoDB connection failed`

    **Solutions:**
    - Verify MongoDB is running locally: `mongod --version`
    - Check your `DB_URL` in the server `.env` file
    - If using MongoDB Atlas, ensure your IP is whitelisted
    - Verify database credentials are correct
  </Accordion>

  <Accordion title="CORS errors">
    **Error:** `CORS policy: No 'Access-Control-Allow-Origin' header`

    **Solutions:**
    - Ensure `CORS_ORIGIN` in server `.env` matches your client URL
    - Default should be `http://localhost:5173`
    - Restart the server after changing environment variables
  </Accordion>

  <Accordion title="Port already in use">
    **Error:** `Port 3000 is already in use`

    **Solutions:**
    - Change the `PORT` in server `.env` to another port (e.g., 3001)
    - Kill the process using port 3000:
      ```bash
      # On macOS/Linux
      lsof -ti:3000 | xargs kill -9

      # On Windows
      netstat -ano | findstr :3000
      taskkill /PID <PID> /F
      ```
  </Accordion>

  <Accordion title="Environment variables not loading">
    **Error:** `process.env.VITE_BACKEND_URL is undefined`

    **Solutions:**
    - Ensure `.env` files are in the correct directories
    - Restart both dev servers after creating `.env` files
    - Vite variables must be prefixed with `VITE_`
    - Don't commit `.env` files to version control
  </Accordion>
</AccordionGroup>
