OrionRides uses a modular CSS approach where each component has its own dedicated stylesheet. The design system is built on CSS custom properties (variables) for theming and follows a mobile-first responsive design philosophy.

## CSS Architecture

### Component-Level Styling

Each component has its own `.css` file co-located with the component:

```
src/components/
├── Navbar.jsx
├── Navbar.css
├── VehicleCards.jsx
├── VehicleCards.css
├── HeroSection.jsx
└── HeroSection.css
```

This modular approach ensures:
- **Scoped styles** - Each component's styles are isolated
- **Easy maintenance** - Styles are located next to their components
- **Better organization** - No massive CSS files

## Responsive Design

### Breakpoint Strategy

The application uses a mobile-first approach with the following breakpoints:

| Breakpoint | Max Width | Target Devices |
|------------|-----------|----------------|
| Desktop XL | 1366px | Large screens |
| Desktop | 1024px | Standard laptops |
| Tablet | 768px | Tablets |
| Mobile | 480px | Phones |

## Design Utilities

### Border Radius Convention

- **Pill shapes**: `border-radius: 100vw`
- **Rounded cards**: `border-radius: 1rem` or `0.7rem`
- **Circular elements**: `border-radius: 50%`

### Shadow Patterns

```css
/* Card shadow */
box-shadow: 0 20px 40px rgba(0, 0, 0, 0.3);

/* Subtle shadow */
box-shadow: 0 10px 20px rgba(0, 0, 0, 0.3);

/* Toast notification */
box-shadow: 0px 0px 1rem rgba(0, 0, 0, 0.3);
```

### Z-Index Layering

| Layer | Z-Index | Usage |
|-------|---------|-------|
| Toast | 1000 | Notifications |
| Header | 999 | Navigation bar |
| Borders | 998 | Decorative borders |
| Menu | -2 | Mobile menu background |

## Best Practices

1. **Use CSS Variables** - Always reference color variables instead of hardcoded values
2. **Mobile-First** - Write base styles for mobile, then add `@media` queries for larger screens
3. **Component Scoping** - Keep styles in component-specific CSS files
4. **Consistent Spacing** - Use rem units for spacing (0.5rem, 1rem, 2rem)
5. **Semantic Class Names** - Use descriptive class names (`.vehicle-actions`, `.profile-modal`)
6. **Smooth Transitions** - Add `transition: all 0.2s` or `0.3s` for interactive elements
