## Rental Status Management

Rentals can have three statuses:

1. **active** - Currently active rental
2. **cancelled** - User cancelled the rental
3. **expired** - Rental period has ended

## Cancellation Flow

### Frontend Implementation

From `client/src/components/ProfileModal.jsx:38-55`:

```javascript
const cancelRental = async (carId) => {
    try {
        let res = await fetch(`${import.meta.env.VITE_BACKEND_URL}/api/cancel-ride/${carId}`, {
            headers: {
                "Content-Type": "application/json",
                "Authorization": `Bearer ${authToken}`,
            },
            method: "POST",
        });
        if (res.ok) {
            userAuthentication();
            showToast("Rental cancelled successfully!", 'success');
        }
    } catch (error) {
        console.log("Error in cancelling the rental: ", error);
        showToast("Failed to cancel rental!", 'error');
    }
};
```

### Backend Cancellation Endpoint

From `server/routes/index.route.js:90-131`:

```javascript
router.post("/cancel-ride/:car_id", authMiddleware, async (req, res) => {
    try {
        let user = req.user;
        let { car_id } = req.params;

        let car = await Car.findOneAndUpdate(
            { _id: car_id },
            { current_owner: null, availability: true },
            { new: true }
        );

        if (!car) {
            return res.status(404).json({
                message: "Car not found!",
            });
        }

        let rentedCar = await RentedCar.findOneAndUpdate(
            { owner: user._id, car: car_id },
            { status: "cancelled" },
            { new: true }
        );

        if (!rentedCar) {
            return res.status(404).json({
                message: "Rental not found for this user and car!",
            });
        }

        res.json({
            message: "Car rental cancelled successfully!",
            car,
            rentedCar,
        });
    } catch (error) {
        console.error("Error:", error);
        res.status(500).json({
            message: "Error cancelling car rental!",
            error: error.message,
        });
    }
});
```

## API Endpoints

### Get User Profile

```
GET /api/user
```

**Headers:**
```
Authorization: Bearer <token>
```

**Response:**
```json
{
  "_id": "...",
  "name": "John Doe",
  "email": "john@example.com",

```

## Features Summary

### Dashboard Features
- View profile information (name, email, avatar)
- Display all rental history (active, cancelled, expired)
- Cancel active rentals
- View rental dates and vehicle details
- Direct link to explore vehicles when no rentals exist
- Logout functionality

### Data Management
- Automatic user authentication on load
- Real-time rental status updates
- Toast notifications for actions
- Optimistic UI updates after cancellation
- Populated car details in rental records
